## Claim Structure

### Base Claims (priority: May 11, 2026 via provisional 64/063,037)
1. **General-purpose sovereign data governance architecture** — not healthcare-specific
2. **Zero-PHI constraint** — claimed broadly as applicable to any regulated data domain
3. **EIP-191 consent primitive** — unified auth + consent + authorization pattern
4. **Sovereign Guide model** — consent-gated practitioner access
5. **Behavioral health as proving ground** — one embodiment, not the only application

### CIP Claims — Phase 5 New Matter (priority: October 2026 filing date)
**Two-Party Wallet-Signed Mutual Authentication with Client-Side Encrypted Session Recording:**

1. **Two-party mutual authentication** — both Sovereign Guide and Participant submit EIP-191 signatures before session start; smart contract verifies both signatures on-chain, creating immutable proof of bilateral consent; neither party can fabricate a session with an unconsenting counterparty
2. **Client-side encrypted recording** — live video encrypted in the browser (AES-256-GCM) before leaving the participant's device; encrypted files pinned to IPFS; only the cryptographic hash recorded on-chain; participant holds sole decryption key; playback requires explicit participant key input with no auto-play capability

**Novel combination:** No existing telehealth platform — including Doxy.me, Daily.co, Zoom — implements two-party wallet-signed mutual consent with client-side encrypted recording and on-chain attestation.
